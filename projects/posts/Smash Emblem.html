Smash Emblem

[ Play on itch.io ]

A strategy game partly inspired by Smash Bros and Fire Emblem, where within a turn-based system you attempt to knock out your opponents from the playing area. The biggest challenge, as well as the focus for this project, was developing the AI for each troop for when you play against an AI opponent. This required me to develop my own state machine in C#, where each state was its own class and handled its own execution, as well as any transitions to the next state. This design ended up being very versatile and easy to expand upon once the foundation was set, but getting to that point required me to devise a way for a state and troop object in the code to communicate with each other, and at the same time to ensure that a given state could only communicate with the troop it corresponded to, while still being able to access relevant information about the state of the game. Some of the solution to this was universal to all states, which lead to a great use case for an abstract parent class, but some of it, like the relevant information a state would need access to, had to be on a case by case basis for each particular state.

If I were to revisit and expand on this project, I would love to go back and design more troop AIs for the player to go up against, leading to something of an entire single-player campaign. I'm quite satisfied with how I set up the underlying system for troop AIs to work, and I think that would make it a really fun project to iterate quickly on.
